# Exp54 / Exp22b Symmetric Fixed-History Template Audit Plan

## Summary

The current fixed-history Exp22 audit is the right repair for the bad raw/no-chat-template result, but it is one-sided: the fixed continuation is generated by the IT-native/chat cell and then replayed through `pt_raw`, `it_native`, and `it_raw`.

This plan adds the missing symmetric arm:

> Generate a PT-raw continuation, replay exactly that same continuation through PT raw, IT native-chat, and IT raw/no-template cells, and compare the convergence-gap signature under the same history.

The goal is not to prove template invariance. The goal is to separate three things that were confounded in the old no-template experiment:

1. **History drift:** PT and IT generate different continuations and lengths.
2. **Prompt serialization:** IT native chat template vs raw text.
3. **Checkpoint dynamics:** whether the IT model stabilizes later than PT on the same token history.

If both IT-teacher and PT-teacher fixed-history audits show a positive native IT convergence gap, the convergence-gap paper becomes much harder to dismiss as a chat-template or free-generation artifact. If only the IT-teacher arm is positive, the paper should be scoped to native IT trajectories. If neither arm is positive, the convergence-gap paper needs major reframing.

## Core Question

At a fixed continuation history, does the IT model remain farther from its own final prediction late in the stack than the PT model?

More concretely, for each teacher source `T`:

- Generate forced continuation tokens with `T`.
- Replay the same forced token IDs through:
  - `pt_raw`: PT checkpoint on raw prompt.
  - `it_native`: IT checkpoint on its native chat/instruction template.
  - `it_raw`: IT checkpoint on raw prompt/no chat template.
- Compute layerwise convergence-gap metrics for each cell.
- Compare:
  - `native_fixed_effect = it_native - pt_raw`
  - `raw_fixed_effect = it_raw - pt_raw`
  - `template_delta = it_native - it_raw`

Primary teacher sources:

| Teacher source | Meaning | Why it matters |
|---|---|---|
| `it_native` | Continuation generated by IT checkpoint in native chat-template regime | Tests convergence gap on natural IT operation while holding history fixed across cells |
| `pt_raw` | Continuation generated by PT checkpoint on raw prompt | Tests whether the IT convergence-gap signature survives on a PT-generated history |

Optional later teacher source:

| Teacher source | Meaning | Status |
|---|---|---|
| `external_text` | Human/reference continuation from a fixed text dataset | Useful but lower priority; needs dataset design |

## Hypotheses

### H1: Free-generation no-template failure was mostly history drift

Prediction:

- `it_raw - pt_raw` is positive under fixed history, even if smaller than `it_native - pt_raw`.
- `it_native - it_raw` is positive.

Interpretation:

- Raw/no-template free generation was too OOD to serve as a clean ablation.
- The convergence-gap signature is partly weight/dynamics-driven and partly strengthened by native prompt serialization.

### H2: Convergence gap is native-trajectory specific

Prediction:

- IT-teacher arm: `it_native - pt_raw` positive.
- PT-teacher arm: weak, mixed, or near zero.

Interpretation:

- The signature is strongest on trajectories the IT model naturally takes.
- The paper should avoid saying the effect is history-invariant.
- Main claim becomes: released IT models settle later under their native interface and natural IT histories.

### H3: Convergence gap is template/trajectory artifact

Prediction:

- IT-teacher arm weak or unstable.
- PT-teacher arm weak or unstable.
- Effects disappear under fixed histories or balanced endpoint controls.

Interpretation:

- The convergence-gap paper is in trouble as a broad NeurIPS paper.
- Late MLP graft/swap evidence may still support a narrower causal-localization claim, but the observational signature should be demoted.

### Prompt Support

Use the same prompt support as the current Exp22 fixed-history audit unless there is a documented reason to change it.

DATASET=data/eval_dataset_v2.jsonl

Do not mix prompt supports inside the headline result. If a shorter support is used for debug/smoke, keep it separate and never paper-facing.

### Teacher Generation

For each prompt and each model, generate one teacher continuation under the specified teacher cell.

Teacher cell definitions:

| Teacher source | Checkpoint | Prompt regime | Prompt construction |
|---|---|---|---|
| `it_native` | IT | native | `_prompt_for_regime(variant="it", prompt_regime="native")` |
| `pt_raw` | PT | raw/native | `_prompt_for_regime(variant="pt", prompt_regime="native")` |

For `pt_raw`, the "native" prompt regime is raw text, because PT has no chat template.

Stop condition:

- Greedy top-1 generation.
- Stop on model-specific stop token IDs.
- Cap at `MAX_NEW_TOKENS=128`.
- Record the exact `forced_ids` and `teacher_generated_text`.

The replay phase must use exactly the same `forced_ids` for all cells in that teacher arm.

### Replay Cells

For each teacher arm:

| Cell | Checkpoint | Prompt regime | Purpose |
|---|---|---|---|
| `pt_raw` | PT | raw/native | PT baseline under fixed history |
| `it_native` | IT | native chat/instruction template | Native released IT interface |
| `it_raw` | IT | raw/no-template | Prompt-serialization stress test |

Important: replay should not allow the cell to choose new tokens. Each cell is teacher-forced through the same `forced_ids`.

## Metrics

### Primary Outcome

Primary convergence-gap outcome:

```text
late_kl_mean
```

This is the late-layer mean KL-to-final readout after skipping the first few generated positions, matching the existing Exp22 analysis convention.

Primary comparisons:

| Name | Formula | Interpretation |
|---|---|---|
| `native_fixed_effect` | `it_native - pt_raw` | IT native interface vs PT on same history |
| `raw_fixed_effect` | `it_raw - pt_raw` | IT weights under raw/no-template vs PT on same history |
| `template_delta` | `it_native - it_raw` | Added effect of native IT prompt serialization |

Run each comparison separately for each teacher source:

```text
it_teacher/native_fixed_effect
it_teacher/raw_fixed_effect
it_teacher/template_delta
pt_teacher/native_fixed_effect
pt_teacher/raw_fixed_effect
pt_teacher/template_delta
```

### Secondary Outcomes

Keep these secondary and diagnostic:

| Outcome | Role |
|---|---|
| `future_top1_flip_rate` | Endpoint-free movement check |
| `adjacent_js_mean` | Endpoint-free layer-to-layer movement check |
| `late_confidence_gap` | Helps identify endpoint confidence artifacts |
| forced-token target rank/logprob | Measures how OOD the forced history is for each cell |
| length/cap rates | Confirms fixed history removed free-generation length drift |

Do not let secondary outcomes override the primary read unless the primary and all endpoint-free checks disagree.

## Quality Gates

Hard gates:

| Gate | Required |
|---|---|
| malformed records | 0 or clearly negligible with documented exclusions |
| missing aligned cell records | 0 for paper-facing support |
| forced token IDs | identical across replay cells within each `(model, teacher_source, prompt_id)` |
| step counts | identical across replay cells within each `(model, teacher_source, prompt_id)` |
| teacher-cell self replay | teacher cell should recover the forced token as top-1 almost always; median forced-token rank should be `1` |
| probe families | raw and tuned both present for paper-facing runs |
| PDF/table claim checker | all new claims audited |

Soft gates:

| Gate | Target | If failed |
|---|---:|---|
| CEM retention | >= 0.70 | Use paired same-prompt-step estimate as primary and CEM as sensitivity |
| max post-match SMD | <= 0.10 | Report imbalance; do not overclaim endpoint-deconfounding |
| top1 match to forced token | high enough for interpretability | Report rank/logprob; treat low match as OOD warning |
| per-family sign consistency | positive in most dense families | Avoid dense-mean-only headline |
| usable forced steps after skip | enough rows per model/teacher/cell | Report short/EOS-heavy arms separately; do not let short continuations dominate |

Current single-model small run had `ok: false` because CEM gates were not clean. That is not fatal, but paper-facing use needs either cleaner matching or clear paired-history framing.

Important analysis rule:

- The **primary fixed-history estimand is paired same-prompt-step**, because the whole point of Exp54 is that token history and step index are already held fixed.
- CEM/endpoint matching is a **sensitivity analysis**, not a hard validity condition for the fixed-history result. If paired effects are clean but CEM retention is low, report CEM as diagnostic/OOD balance rather than failing the experiment.
- The teacher-source self-replay check is different from ordinary OOD diagnostics: for `it_native` teacher, the `it_native` replay should nearly reproduce the greedy teacher choices; for `pt_raw` teacher, the `pt_raw` replay should nearly reproduce them. Failure here indicates a generation/replay mismatch, tokenizer/cache bug, or mask mismatch.

## Interpretation Rules

### Strong result

Pattern:

- `it_native - pt_raw` positive under both teacher sources.
- raw fixed effect is positive or at least not strongly reversed.
- template delta is positive.
- Per-family signs mostly positive.

Paper wording:

> Fixed-history audits show that the later-stabilization signature is not a free-generation length artifact. It remains visible when PT and IT are replayed on the same token histories, including histories generated by either checkpoint, while native IT formatting strengthens the effect.

### Moderate result

Pattern:

- IT-teacher positive.
- PT-teacher weak/mixed.
- Template delta positive.

Paper wording:

> The fixed-history audit supports a native-trajectory interpretation: the convergence gap is strongest on histories generated by the instruction-tuned model under its released interface. PT-generated histories give weaker evidence, so we do not claim history-invariance.

### Weak result

Pattern:

- Both teacher sources weak/mixed after endpoint matching.

Paper action:

- Do not use Exp22 as a strong main-text pillar.
- Keep convergence gap as observational and scope it tightly.
- Rely more on late MLP graft/swap localization, or delay submission until better evidence exists.

### Bad result

Pattern:

- Native fixed effect reverses across multiple families.
- Endpoint-free checks do not support remaining movement.

Paper action:

- Do not submit the convergence-gap paper in its current form.
- Reframe around coarse late-MLP intervention effects only, or merge into background/appendix of another work.

## Red-Team Concerns Addressed

### Concern 1: No-template result reversed, so the finding is a chat-template artifact

Response if successful:

- Old no-template was free-running and changed histories/lengths.
- Fixed-history replay keeps token IDs identical.
- Raw/no-template under fixed history can be reported separately from native-template effects.

### Concern 2: IT-teacher history bakes in the IT result

Response if PT-teacher arm succeeds:

- The effect also appears on PT-generated histories.
- Therefore it is not only an IT-trajectory artifact.

Response if PT-teacher arm is mixed:

- Scope to native IT trajectories.
- Do not claim history invariance.

### Concern 3: Endpoint matching is not clean

Response:

- Report paired same-prompt-step estimates as the primary fixed-history result.
- Use CEM/endpoint matching as sensitivity.
- Explicitly show CEM retention/SMD.

### Concern 4: Forced continuations are OOD for the non-teacher model

Response:

- Report forced-token rank/logprob and top1 match rates.
- Analyze early steps separately if late forced steps become too OOD.
- If PT-teacher and IT-teacher disagree sharply, present that as trajectory dependence.
